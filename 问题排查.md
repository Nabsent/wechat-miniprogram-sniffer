# 🔧 问题排查指南

## 问题：代理配置了，能上网，但抓不到小程序的请求

这是**最常见**的问题，原因是微信小程序的请求都是 **HTTPS 加密**的，普通代理无法解密查看内容。

---

## 🎯 快速诊断

### 步骤 1: 使用调试版代理

停止原来的代理，运行调试版：

```cmd
python proxy_sniffer_debug.py
```

调试版会显示**所有**经过的请求，即使是 HTTPS 也会显示域名。

### 步骤 2: 观察日志

打开微信小程序后，应该能看到类似这样的日志：

```
[1] HTTPS: servicewechat.com:443
    ⚡ 微信相关: servicewechat.com
[2] HTTPS: res.wx.qq.com:443
    ⚡ 微信相关: res.wx.qq.com
[3] HTTPS: api.weixin.qq.com:443
    ⚡ 微信相关: api.weixin.qq.com
```

### 步骤 3: 判断情况

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| **没有任何日志** | 代理配置有问题 | 见下方"情况A" |
| **有HTTPS请求** | 流量正常，但被加密了 | 见下方"情况B" |
| **有HTTP请求** | 部分流量未加密 | 应该能捕获到 |

---

## 情况 A: 完全没有日志输出

说明手机流量没有经过代理，检查：

### 1. 测试代理连接

```cmd
python test_proxy.py
```

应该看到：
```
✅ 代理服务器工作正常！
✅ 可以使用网络端口
```

### 2. 检查手机浏览器

手机浏览器访问 `http://www.baidu.com`，电脑应该立即有日志：

```
[1] HTTP: GET http://www.baidu.com/
```

如果浏览器有日志，但微信没有 → 跳到"情况B"

如果浏览器也没有日志：

#### 检查清单：

- [ ] 代理服务器在运行中（没有崩溃）
- [ ] 手机和电脑在同一WiFi
- [ ] 手机代理IP地址正确（运行 `python get_ip.py` 确认）
- [ ] 手机代理端口是 `8888`
- [ ] 手机代理设置为"手动"，不是"自动"
- [ ] Windows防火墙允许端口8888

#### 防火墙设置（Windows）：

```cmd
# 临时关闭防火墙测试（测试后记得打开）
netsh advfirewall set allprofiles state off

# 或者添加规则允许 8888 端口
netsh advfirewall firewall add rule name="Proxy 8888" dir=in action=allow protocol=TCP localport=8888
```

## 情况 B: 有日志但都是HTTPS，抓不到文件

这是**正常的**，微信小程序使用完全加密的HTTPS通信。

### 解决方案：

#### 方案 1: 使用专业抓包工具（推荐）

普通代理无法解密HTTPS，需要安装证书。推荐使用：

**Windows: Fiddler Classic (免费)**

1. 下载安装：https://www.telerik.com/fiddler/fiddler-classic
2. 安装证书：
   - Tools → Options → HTTPS → 勾选 "Decrypt HTTPS traffic"
   - 点击 "Actions" → "Export Root Certificate to Desktop"
3. 手机安装证书：
   - 将证书发送到手机
   - 设置 → 安全 → 安装证书 → 选择证书文件
   - Android需要信任用户证书
4. 手机配置代理为电脑IP:8888（Fiddler默认端口）

**macOS: Charles (付费，可试用)**

配置方法类似Fiddler。

#### 方案 2: 使用 mitmproxy（免费，命令行）

```cmd
pip install mitmproxy

# 启动代理
mitmdump -p 8888 --set block_global=false

# 手机访问 http://mitm.it 下载证书
```

#### 方案 3: 反编译小程序（高级）

使用工具解包小程序，直接查看代码中的API地址。

#### 方案 4: 监控网络流量（抓包文件URL）

有些小程序的文件URL会在页面HTML或JS中：

```cmd
# 使用调试版，查看HTTPS域名
python proxy_sniffer_debug.py

# 如果看到文件相关域名，记录下来
# 然后使用开发者工具或抓包看请求参数
```

## 情况 C: 有HTTP请求但没捕获到文件

可能原因：

### 1. 文件是通过异步加载的

刷新页面或重新触发下载。

### 2. 文件扩展名不在识别列表

编辑 `proxy_sniffer_debug.py`，添加扩展名：

```python
self.file_extensions = [
    '.pdf', '.doc',
    '.custom_ext',  # 添加你的扩展名
]
```

### 3. 使用自定义Content-Type

同样在代码中添加：

```python
self.file_content_types = [
    'application/pdf',
    'application/custom-type',  # 添加自定义类型
]
```

### 4. 文件通过POST请求

当前版本主要捕获GET请求，如果是POST，需要修改代码：

```python
# 在 handle_http_request 中
if method in ['GET', 'POST']:  # 原来可能只有GET
    # ... 处理逻辑
```

## 🔍 高级调试

### 查看完整请求头

修改 `proxy_sniffer_debug.py`，在处理函数中添加：

```python
print("完整请求头:")
print(request_str[:500])  # 打印前500字符
```

### 保存所有请求

在 `handle_http_request` 末尾添加：

```python
# 保存所有HTTP请求
with open('all_requests.log', 'a', encoding='utf-8') as f:
    f.write(f"\n{'='*70}\n")
    f.write(f"[{datetime.now()}] {method} {url}\n")
    f.write(request_str)
```

### 检查响应头

```python
if response:
    print("响应头:")
    header_end = response.find(b'\r\n\r\n')
    if header_end > 0:
        print(response[:header_end].decode('utf-8', errors='ignore'))
```

## 📱 特殊情况：微信小程序的限制

### Android 7.0+

Android 7.0+ 默认不信任用户安装的证书，小程序无法抓包。

**解决方案：**
1. 使用 Android 6.0 或更低版本的手机/模拟器
2. Root手机，将证书安装为系统证书
3. 使用Xposed框架绕过证书检查

### iOS 10.3+

需要在"设置 → 通用 → 关于本机 → 证书信任设置"中启用证书。

### 小程序使用了证书绑定（Certificate Pinning）

部分小程序会验证服务器证书，无法用中间人方式抓包。

**解决方案：**
1. 使用Frida等工具绕过证书校验
2. 反编译小程序获取API接口

## ✅ 推荐的完整抓包流程

### 对于普通小程序（未加固）：

1. **安装 Fiddler** (Windows) 或 **Charles** (macOS)
2. **安装并信任证书**
3. **配置手机代理**
4. **打开小程序，操作文件下载**
5. **在Fiddler中筛选文件请求**（按文件类型或大小过滤）
6. **右键 → Save → Response Body** 保存文件
7. **或复制URL，使用我们的下载工具**：
   ```cmd
   python file_downloader.py --url "https://..."
   ```

### 对于HTTP混合的小程序：

使用我们的调试版代理：

```cmd
python proxy_sniffer_debug.py
```

能捕获HTTP部分的文件。

## 🆘 仍然无法解决？

### 提供以下信息以便诊断：

1. 运行 `python proxy_sniffer_debug.py` 的完整输出
2. 运行 `python test_proxy.py` 的结果
3. 手机浏览器访问百度时，代理是否有日志
4. 小程序名称（如果方便透露）
5. 点击文件时，手机上发生了什么（是否打开、是否下载等）

### 替代方案：

如果实在无法抓包，可以尝试：

1. **查看小程序开发者工具**：
   - 用微信开发者工具打开小程序
   - 可以看到网络请求

2. **联系小程序开发者**：
   - 询问文件下载接口

3. **使用手机抓包应用**：
   - Android: Packet Capture (不需要Root)
   - iOS: Stream (需要越狱或企业签名)
